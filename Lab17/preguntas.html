Preguntas

  • ¿Qué ventajas tiene escribir el código SQL únicamente en la capa del modelo?
    proporciona una separación clara de responsabilidades, promueve la reutilización del código, facilita la abstracción de la base de datos, mejora la seguridad y simplifica las pruebas. Esto permite un desarrollo más eficiente y una aplicación más mantenible y segura en el tiempo.
  • ¿Qué es SQL injection y cómo se puede prevenir?
    es una vulnerabilidad de seguridad que ocurre cuando un atacante inserta código SQL malicioso en las consultas de entrada de una aplicación, permitiéndole manipular la base de datos de forma no autorizada. Para prevenirlo, es crucial utilizar consultas parametrizadas o sentencias preparadas, validar y sanitizar los datos de entrada, limitar los privilegios de la base de datos y mantener la seguridad en todas las capas de la aplicación.
